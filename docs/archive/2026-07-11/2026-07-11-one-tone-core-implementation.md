# One-Tone 阶段 1–2 核心实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立一个使用 `uv` 运行的 Python 核心工具，完成 Palette、Plan、Preview、Apply、Snapshot、Verify 和指定事务 Rollback 的可测试闭环。

**Architecture:** 纯函数 Palette 和带确定性 Hash 的 Plan 位于核心数据层；Adapter 通过统一结构化结果提供目标读写能力；TransactionStore 负责事务目录、快照、状态和恢复，CLI 负责编排命令和目标选择。真实 Windows/Terminal/编辑器/Codex/Chrome 适配器本阶段只返回 `skipped`，文件型 Adapter 只作为本地验证目标。

**Tech Stack:** Python 3.11+、标准库、pytest、uv、`argparse`、JSON、SHA-256。

## Global Constraints

- 使用 Python。
- 使用 `uv` 管理依赖、运行程序和测试。
- Preview 只检测、生成 Palette、构建 Plan、验证并保存 Plan，不修改配置。
- Apply 只接受已有 `plan_id`，执行前必须验证 Plan Hash。
- 每次 Apply 创建独立 transaction 目录并保存原始配置备份；没有成功 Snapshot 的目标不得 Apply。
- Rollback 必须通过明确的 `transaction_id` 执行，只恢复该事务保存的备份。
- Adapter 统一返回 `AdapterResult(target, status, changed, verified, message)`。
- 状态只使用 `PENDING`、`APPLIED`、`PARTIAL`、`FAILED`、`ROLLED_BACK`。
- 关键 foreground/background、accent foreground/accent、selection foreground/selection background 的对比度目标为 `>= 7:1`。
- 未知目标必须拒绝或标记为 `skipped`，不得猜测兼容性。
- 失败时自动回滚本次已修改目标，并保留清晰结果。
- 不引入数据库、后台服务、事件溯源、复杂状态机或依赖注入框架。

def load_plan(plan_id: str, plans_dir: Path) -> Plan:
    payload = json.loads((plans_dir / f"{plan_id}.json").read_text(encoding="utf-8"))
    expected = payload.pop("hash", "")
    actual = compute_plan_hash(payload)
    if expected != actual:

    payload["hash"] = expected
    payload["targets"] = tuple(payload["targets"])
    return Plan(**payload)

from one_tone.adapters import AdapterResult, FileAdapter
from one_tone.plan import create_plan
from one_tone.transaction import TransactionStatus, TransactionStore, apply_plan


def test_apply_creates_isolated_transaction_and_rollback_restores_only_it(tmp_path):
    config = tmp_path / "theme.json"
    config.write_text('{"theme": "original"}', encoding="utf-8")
    plan = create_plan("#7C3AED", ["file-demo"], plan_id="plan-test-004")
    store = TransactionStore(tmp_path / "transactions")
    adapter = FileAdapter("file-demo", config)

    record = apply_plan(plan, {"file-demo": adapter}, store, confirm=True)
    assert record.status == TransactionStatus.APPLIED
    assert (store.path_for(record.id) / "backup" / "file-demo.json").exists()
    assert config.read_text(encoding="utf-8").find("plan-test-004") >= 0

    restored = store.rollback(record.id, {"file-demo": adapter})
    assert restored.status == TransactionStatus.ROLLED_BACK
    assert config.read_text(encoding="utf-8") == '{"theme": "original"}'


def test_apply_failure_automatically_rolls_back_successful_targets(tmp_path):
    first = tmp_path / "first.json"
    second = tmp_path / "second.json"
    first.write_text('{"theme": "first"}', encoding="utf-8")
    second.write_text('{"theme": "second"}', encoding="utf-8")
    plan = create_plan("#7C3AED", ["first", "second"], plan_id="plan-test-005")
    store = TransactionStore(tmp_path / "transactions")

    class FailingVerifyAdapter(FileAdapter):
        def verify(self, plan):
            return AdapterResult(self.target, "failed", True, False, "forced verify failure")

    adapters = {
        "first": FileAdapter("first", first),
        "second": FailingVerifyAdapter("second", second),
    }
    record = apply_plan(plan, adapters, store, confirm=True)

    assert record.status == TransactionStatus.FAILED
    assert first.read_text(encoding="utf-8") == '{"theme": "first"}'


def test_apply_marks_mixed_success_and_skipped_targets_partial(tmp_path):

    config.write_text('{"theme": "original"}', encoding="utf-8")
    plan = create_plan("#7C3AED", ["file-demo", "codex"], plan_id="plan-test-006")
    store = TransactionStore(tmp_path / "transactions")
    adapters = {
        "file-demo": FileAdapter("file-demo", config),
        "codex": UnsupportedAdapter("codex"),
    }

    record = apply_plan(plan, adapters, store, confirm=True)

    assert record.status == TransactionStatus.PARTIAL

- [ ] **Step 3: Write minimal implementation**

Create a unique transaction ID using UTC timestamp plus a short counter/UUID suffix. Before Apply, require `confirm=True`; call Detect, Snapshot, Apply, Verify in order per target. Skip targets whose Detect result is `skipped`; if at least one target is skipped and all non-skipped targets succeed, mark the transaction `PARTIAL` and keep successful changes.
